# 테스트 계획

## 개요

본 테스트 계획은 MVP 요구사항이 실제 사용자 테스트에 필요한 수준으로 동작하는지 확인하는 것을 목표로 한다.

## 기능 테스트

| 테스트 ID | 관련 요구사항 | 테스트 내용 | 기대 결과 |
| --- | --- | --- | --- |
| TC-001 | REQ-001 | 사용자가 메시지를 입력하고 응답을 확인한다. | 채팅 UI에 사용자 메시지와 응답이 순서대로 표시된다. |
| TC-002 | REQ-002, REQ-003 | 특정 메시지에서 브랜치를 생성한다. | 새 브랜치가 생성되고 원본 메시지와의 분기 관계가 저장된다. |
| TC-003 | REQ-004 | 브랜치가 포함된 대화를 그래프로 확인한다. | 브랜치 관계가 노드와 엣지로 표시된다. |
| TC-004 | REQ-005 | 그래프에서 브랜치를 클릭한다. | 선택한 브랜치의 대화 내용으로 이동한다. |
| TC-005 | REQ-006, REQ-007 | 브랜치를 비활성화한다. | 해당 브랜치가 삭제되지 않고 낮은 강조 상태로 표시된다. |
| TC-006 | REQ-008 | 숨김 또는 접기 처리된 브랜치를 다시 표시한다. | 사용자가 브랜치를 다시 확인하거나 활성화할 수 있다. |
| TC-007 | REQ-009 | 그래프를 확대하고 축소한다. | 그래프 배율이 변경되며 주요 노드 식별이 가능하다. |
| TC-008 | REQ-010 | 브랜치에 마우스를 올린다. | 브랜치 요약 미리보기가 표시된다. |
| TC-009 | REQ-011 | Mock LLM과 실제 LLM 연동 지점을 교체한다. | 채팅 UI 변경 없이 응답 제공 모듈만 교체 가능하다. |

## 사용성 테스트

| 테스트 ID | 목적 | 테스트 내용 | 측정 항목 |
| --- | --- | --- | --- |
| UT-001 | 브랜치 생성 이해도 확인 | 사용자가 긴 답변에서 꼬리질문용 브랜치를 생성한다. | 브랜치 생성 성공 여부, 완료 시간, 혼란 지점 |
| UT-002 | 원래 목적 복귀 가능성 확인 | 사용자가 브랜치 탐색 후 원래 대화 흐름으로 돌아온다. | 복귀 성공 여부, 클릭 수, 주관적 난이도 |
| UT-003 | 그래프 가독성 확인 | 사용자가 여러 브랜치 중 특정 브랜치의 분기 지점을 찾는다. | 탐색 시간, 오답률, 만족도 |
| UT-004 | 비활성화 기능 효용 확인 | 사용자가 불필요한 브랜치를 비활성화하고 화면 혼잡도를 평가한다. | 기능 사용 여부, 전후 만족도 |

## 성공 기준 검증

- 사용자 만족도 조사에서 브랜치 기능과 시각화 UI의 유용성을 확인한다.
- 브랜치 시각화 UI 사용 전후의 이용 시간과 브랜치 생성 횟수를 비교한다.
- 사용자가 본래 대화 목적을 잃지 않는다고 응답하는 비율을 측정한다.
- 실제 사용자 테스트 결과와 정성 피드백을 `docs/review_notes.md`에 기록한다.
