# 리뷰 기록

## 2026-05-15 구조 개편 기록

### 확인 내용

- 기존 루트에는 `00_brief.md`, `01_questions.md`, `02_answers.md`, `03_prd.md`, `AGENTS.md`, `commit_log.md`가 존재했다.
- 요청된 목표 구조는 `docs/` 하위에 PRD, 요구사항, 구현 계획, 테스트 계획, 요구사항 상태, 리뷰 기록을 배치하는 형태이다.
- 기존 `03_prd.md`는 새 구조의 원본 기획서 역할을 하므로 `docs/PRD.md`로 이동했다.

### 이관 내용

- `00_brief.md`의 프로젝트 브리프 내용은 `docs/PRD.md`와 `docs/requirements.md`의 기준 정보로 반영했다.
- `01_questions.md`와 `02_answers.md`의 인터뷰 흐름은 본 리뷰 기록의 근거로 보존했다.
- 요구사항 ID 목록은 `docs/requirements.md`로 새로 정리했다.
- 구현 순서는 `docs/implementation_plan.md`로 정리했다.
- 검증 방식은 `docs/test_plan.md`로 정리했다.
- 요구사항별 현재 상태는 `docs/requirement_status.md`로 분리했다.

### PRD 검토 메모

- 문제, 타깃 사용자, 현재 불편, 제안 솔루션, 핵심 기능, MVP 범위, 성공 기준은 현재 문서에 포함되어 있다.
- 그래프 UI의 구체 레이아웃은 아직 확정되지 않았다.
- 실제 LLM API 연동 수준과 Mock 응답 사용 범위는 구현 전 추가 결정이 필요하다.
- 비활성 브랜치의 복구 방식과 위치는 아직 미정이다.
- 성공 기준의 정량 목표값은 아직 확정되지 않았다.

## 기존 인터뷰 요약

### 1차 답변 요약

- 프로젝트는 LLM과의 대화형 채팅 서비스에서 브랜치 기능과 시각화를 통해 채팅 집중도를 향상하는 서비스이다.

### 2차 답변 요약

- 주요 사용자는 ChatGPT를 심도 있게 사용하는 학생, 개발자, 기획자, 연구자이다.
- 핵심 문제는 꼬리질문 누적으로 인한 본래 목적 이탈과 컨텍스트 혼선이다.
- 차별점은 단순 fork가 아니라 브랜치 분기 지점과 관계를 쉽게 파악하게 하는 시각화 UI이다.

### 3차 답변 요약

- MVP 핵심 기능은 기본 LLM 채팅, 메시지 단위 브랜치 생성, 브랜치 분기 지점 시각화, 브랜치 비활성화이다.
- 브랜치 시각화는 그래프 UI가 우선 후보이다.
- 성공 기준은 사용자 만족도, 서비스 이용 트래픽, 서비스 이용 시간 비교이다.

### 4차 답변 요약

- 초기 서비스 형태는 독립 웹 서비스이다.
- MVP는 실제 LLM API 완전 연동보다 브랜치 생성과 그래프 시각화 UI 검증을 우선한다.
- 그래프 UI에는 브랜치 클릭 이동, 비활성화 토글, 확대 및 축소, 마우스 오버 요약 표시가 필요하다.

### 5차 답변 요약

- 브랜치 요약은 LLM 자동 요약 방식이 적합하다.
- 비활성화된 브랜치는 접기 방식으로 숨김 처리할 수 있어야 한다.
- 프로젝트 목표는 데모가 아니라 실제 사용자를 모집할 수 있는 완성도 있는 서비스이다.
