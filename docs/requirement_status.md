# 요구사항 충족 여부

## 개요

본 문서는 `docs/requirements.md`의 요구사항별 구현 상태를 추적한다. 현재는 프로젝트 구조 정리 단계이므로 모든 기능 요구사항은 미구현 상태이다.

## 상태 기준

| 상태 | 의미 |
| --- | --- |
| 미착수 | 아직 구현을 시작하지 않았다. |
| 진행 중 | 구현 또는 설계가 진행 중이다. |
| 충족 | 요구사항이 구현되고 기본 검증을 통과했다. |
| 보류 | MVP 범위 또는 의사결정 문제로 대기 중이다. |

| ID | 요구사항 | 상태 | 비고 |
| --- | --- | --- | --- |
| REQ-001 | 기본 LLM 채팅 인터페이스 | 미착수 | 기술 스택 확정 후 진행한다. |
| REQ-002 | 메시지 단위 브랜치 생성 | 미착수 | 데이터 모델 정의가 선행되어야 한다. |
| REQ-003 | 브랜치 분기 지점 기록 | 미착수 | 메시지와 브랜치 관계 설계가 필요하다. |
| REQ-004 | 그래프 기반 브랜치 시각화 | 미착수 | 그래프 라이브러리 또는 직접 구현 여부를 결정해야 한다. |
| REQ-005 | 브랜치 클릭 이동 | 미착수 | 라우팅 또는 상태 전환 방식 결정이 필요하다. |

| REQ-008 | 비활성 브랜치 복구 가능성 | 미착수 | 복구 UI의 위치와 흐름이 미정이다. |
| REQ-009 | 그래프 확대 및 축소 | 미착수 | 그래프 UI 구현 단계에서 검증한다. |
| REQ-010 | 브랜치 요약 미리보기 | 미착수 | Mock 요약부터 시작할 수 있다. |
| REQ-011 | LLM API 연동 고려 구조 | 미착수 | 응답 제공 모듈의 인터페이스 설계가 필요하다. |
| REQ-012 | 사용자 테스트 가능 품질 | 미착수 | MVP 사용 흐름이 완성된 뒤 검증한다. |
